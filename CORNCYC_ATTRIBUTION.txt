CornCyc / Plant Metabolic Network (PMN) Attribution
=============================================================================

The Maysquery pipeline can be configured by the end user to read the CornCyc
Pathway/Genome Database (PGDB) for the curated discovery lane and the maize
pathway-context section of its reports. CornCyc is published by the Plant
Metabolic Network (PMN) under a license that permits redistribution and
modification of derived data, subject to attribution requirements:

  > License includes right to modify and to redistribute the Databases,
  > provided that any modified copies of the Databases, or websites from
  > which such modified copies may be obtained, must clearly and
  > prominently (i) identify the Database(s) from which they were
  > derived; (ii) include all applicable copyright notices and author
  > lists from the Database(s) from which they were derived; and
  > (iii) identify or summarize all modifications that were made.

This file satisfies clauses (i), (ii), and (iii) for any Maysquery output
that contains CornCyc-derived data. It is also referenced from the user-
facing reports (HTML + CSV) when those reports include such data.

Maysquery itself does NOT redistribute the CornCyc flatfiles in its source
repository — `corncyc/` is excluded by .gitignore. The end user installs the
PMN-licensed PGDB locally per the instructions in INSTALLATION.md.

-----------------------------------------------------------------------------
(iii) MODIFICATIONS MADE BY MAYSQUERY
-----------------------------------------------------------------------------
Maysquery does not edit the CornCyc flatfiles in place. The "modifications"
described below are the transformations Maysquery applies in-memory before
re-presenting CornCyc-derived data in its own outputs. The original
CornCyc data on disk remains unchanged.

1. PARSING ONLY (no semantic change)
   `backend/corncyc_loader.py` reads five files from the CornCyc PGDB:
       compounds.dat   reactions.dat   enzrxns.dat   proteins.dat   pathways.dat
   The Pathway-Tools attribute-value records are parsed into Python
   dictionaries indexed by UNIQUE-ID. No values are altered during parsing.

2. CHEBI CROSS-REFERENCE INDEXING
   Compounds are additionally indexed by their `DBLINKS (CHEBI ...)`,
   `DBLINKS (PUBCHEM ...)`, and InChI-Key cross-references to enable
   lookup from a user-supplied ChEBI ID. Cross-reference values are
   reproduced verbatim from the CornCyc record.

3. CHEBI CONJUGATE-FORM EXPANSION (query side, not on CornCyc itself)
   For a user query compound, Maysquery resolves the supplied ChEBI ID to
   the full conjugate-acid/base family via the EBI OLS ontology. Each
   form is then used to look up CornCyc compound frames. The CornCyc
   compound records themselves are not modified — only the *query* is
   expanded before the lookup.

4. MAIZE GENE-ID CASE NORMALISATION
   CornCyc stores B73 NAM v5 maize gene IDs in upper-case
   (e.g. `ZM00001EB117970`). Maysquery normalises these to the
   conventional lower-case prefix (`Zm00001eb117970`) for visual
   consistency with MaizeGDB / Gramene displays. The numeric portion is
   unchanged.

5. DERIVED PRESENTATION
   Maysquery's HTML and CSV reports display the following CornCyc-sourced
   fields verbatim:
     * Compound COMMON-NAME, SYNONYMS, DBLINKS values
     * Reaction COMMON-NAME, EC-NUMBER, LEFT/RIGHT compound participants
     * Pathway COMMON-NAME, SYNONYMS, REACTION-LIST
     * Protein GENE associations (after case normalisation, point 4)
   Each pathway shown in a Maysquery report includes a hyperlink to the
   corresponding canonical PMN page at
       https://pmn.plantcyc.org/pathway?orgid=CORN&id=<PathwayID>
   where the full citation list and curator attributions are displayed.

6. NOT REPRODUCED IN MAYSQUERY OUTPUTS
   The following CornCyc fields are read but not currently surfaced in
   Maysquery's UI or reports:
     * CITATIONS (per-record evidence codes and literature references)
     * COMMENT (long-form curator notes)
     * EXPLANATION-CODE
     * GIBBS-0, ATOM-CHARGES, ATOM-MAPPINGS
   Users requiring the full provenance for a derived annotation should
   consult the CornCyc flatfiles on their local install, or the canonical
   PMN PlantCyc web pages linked from each Maysquery pathway row.
